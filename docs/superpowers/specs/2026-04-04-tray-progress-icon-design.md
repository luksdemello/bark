# Tray Progress Icon — Design Spec

**Date:** 2026-04-04

## Overview

During a file upload (drag-drop), the macOS menu bar icon currently shows no feedback. This spec adds a progress ring to the tray icon — mirroring the in-app `DogIcon` progress ring — using pre-generated PNG frames swapped by the Rust backend in response to events emitted from the frontend.

---

## Architecture

```
useUpload (frontend)
  └─ emits "upload-progress" { progress: number }
        │
        ▼
lib.rs listen handler (backend)
  └─ maps progress → bucket (10, 20, …, 100, 0)
  └─ calls tray.set_icon(tray_progress_<bucket>.png)
        │   or tray_normal.png when progress = 0
```

---

## Components

### 1. PNG icon set

- Files: `src-tauri/icons/tray_progress_10.png` through `tray_progress_100.png` (10 files)
- Each is the same dog face as `tray_normal.png` overlaid with a partial arc (stroke-dasharray technique) matching the in-app ring style
- Generated by a one-off script (`scripts/generate_tray_progress_icons.mjs`) using `sharp` or a Node SVG-to-PNG pipeline
- Size: same as existing tray icons (match `tray_normal.png` dimensions, typically 32×32 or 44×44 @2x)
- Color: `#0a84ff` (same as `ring-fill` in `Icons.tsx`)

### 2. Frontend — `useUpload.ts`

- After each `setProgress(current)` call, emit `"upload-progress"` with `{ progress: current }` via `@tauri-apps/api/event` `emit()`
- After the reset (when `setFilename(null)` + `setProgress(0)`), also emit `"upload-progress"` with `{ progress: 0 }` so the backend reverts to the normal icon

No other changes to `useUpload.ts`.

### 3. Backend — `lib.rs`

Inside `.setup()`, after the tray is built, register a global listener:

```rust
app.listen("upload-progress", move |event| {
    // deserialize { progress: u8 }
    // bucket = round to nearest 10, clamp 0..=100
    // if bucket == 0 → tray_normal.png
    // else           → tray_progress_<bucket>.png
    // tray.set_icon(...)
});
```

The listener holds a clone of the `AppHandle` to resolve the tray by id (`"bark-tray"`).

The reset back to `tray_normal.png` happens when the frontend emits `progress: 0` — no extra timer needed in the backend.

---

## Data Flow

1. User drops a file → `tauri://drag-drop` fires
2. `useUpload` starts a `setInterval` incrementing `progress` by 5 every 100ms
3. Each tick: `setProgress(current)` + `emit("upload-progress", { progress: current })`
4. Backend listener receives the event, picks the right PNG, calls `set_icon`
5. At 100%: frontend clears interval, waits 500ms, then resets `progress` to 0 and emits `{ progress: 0 }`
6. Backend receives 0 → restores `tray_normal.png`

---

## Error Handling

- If the tray can't be found by id, the listener silently returns (matching existing `tray_animation.rs` pattern)
- If JSON deserialization fails, the listener silently returns (no crash)
- PNG files are embedded at compile time with `tauri::include_image!` — missing files are compile errors, not runtime errors

---

## Out of Scope

- Generating icons dynamically in Rust (deferred)
- Showing progress in the Dock badge or window progress bar
- Persisting icon state across app restarts
